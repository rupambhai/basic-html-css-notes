Git is a version control system that tracks changes in your code and helps you collaborate with others.

1. git init

Purpose: Initialize a new Git repository in your project folder. 

			git init

This creates a hidden .git/ folder and starts tracking your project locally.

2. git add .

Purpose: Add all files (in current directory and below) to the staging area. 

			git add .

This prepares the files to be committed. The dot . means ‚Äúall files‚Äù.

3. git config -l

Purpose: List all your Git configuration settings. 

			git config -l

You‚Äôll see settings like user.name, user.email, etc. 

			git config --global user.name "Your Name"
            git config --global user.email "your@email.com"

4. git commit -m "first commit"

Purpose: Save a snapshot of your staged changes to the Git history with a message. 

			git commit -m "first commit"

The -m lets you write a commit message inline.

5. git branch -M main

Purpose: Rename the current branch to main. 

			git branch -M main

Best practice: Use main as the default branch instead of master.

6. git remote add origin <repo-url>

Purpose: Link your local repo to a remote repository (like GitHub). 

			git remote add origin https://github.com/your-username/your-repo.git

This sets origin as a shortcut to your remote GitHub repo. 

			git push -u origin main
